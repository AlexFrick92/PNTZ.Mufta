# Инструкция: Работа с системой стилей приложения

## Обзор системы стилей

Система стилей разделена на пять категорий ресурсов:

1. **AppLabels** - Текстовые строки интерфейса
2. **AppColors** - Цвета и кисти
3. **AppFonts** - Размеры и семейства шрифтов
4. **AppLayouts** - Размеры разметки (ширины, высоты) 
5. **AppSettings** - Общие настройки (числа, строки, флаги)

### Архитектура

Каждый компонент состоит из трех частей:

1. **Styles/{Name}.xaml** - дефолтные значения (компилируются в сборку)
2. **Config/{Name}.xaml** - переопределения для кастомизации (опционально)
3. **Styles/{Name}.cs** - C# адаптер для типобезопасного доступа из кода

## Общий алгоритм добавления нового ресурса

### Шаг 1: Добавить ресурс в XAML

Откройте соответствующий файл `Styles/App{Type}.xaml` и добавьте новый ресурс.

**Правила именования ключа:**
- Формат: `{Окно}_{Элемент}[_{Свойство}]`
- Примеры: `JointProcess_Title`, `Chart_LimitMin_FontSize`, `Recipe_SaveButton`

### Шаг 2: Добавить свойство в C# адаптер

Откройте `Styles/App{Type}.cs` и добавьте новое свойство:

```csharp
/// <summary>Описание назначения ресурса</summary>
public static {ReturnType} YourKey_Name => Get{Type}("YourKey_Name");
```

**Обратите внимание:**
- Имя свойства должно совпадать с ключом из XAML
- Добавьте XML-комментарий с описанием
- Разместите в соответствующую секцию (структура секций должна совпадать в XAML и CS)

### Шаг 3: Использование

#### В C# (ViewModel или Code-Behind):
```csharp
using PNTZ.Mufta.TPCApp.Styles;

// Использование
var value = App{Type}.YourKey_Name;
```

#### В XAML (Binding):
```xml
<Element Property="{DynamicResource YourKey_Name}" />
```

## Переопределение ресурсов (опционально)

Для кастомизации под конкретную установку используйте файлы в папке `Config/`:

1. Откройте `Config/App{Type}.xaml`
2. Раскомментируйте или добавьте переопределение:

```xml
<sys:String x:Key="YourKey_Name">Новое значение</sys:String>
```

Это значение будет использоваться вместо дефолтного из `Styles/App{Type}.xaml`.

**ВАЖНО:** Не изменяйте файлы в `Styles/`, используйте только `Config/` для переопределений.

## Структура секций

В каждом файле XAML существуют следующие секции:
1. Общие
2. Окно создания рецепт
3. Окно процесса свинчивания
4. Окно просмотра результатов
